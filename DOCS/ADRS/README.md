# ADRs (Architecture Decision Records)

This folder is the canonical home for CHESSCHAT architecture decisions.

## Convention
- File name format: `NNNN-short-title.md` (example: `0001-single-nat-gateway.md`).
- Keep one decision per file.
- Use concise, interview-ready rationale with explicit tradeoffs.

## Suggested ADR Sections
- Title
- Status (`Proposed`, `Accepted`, `Superseded`)
- Context
- Decision
- Consequences
- Alternatives Considered

## Scope
Store AWS/Terraform/security/observability/HA decisions here as they are made.
