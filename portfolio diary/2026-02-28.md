### Thought Process and Tradeoffs
- Built Cognito and IAM together because ECS task definitions require role ARNs, and auth choices inform application permission boundaries.
- Scoped data-access permissions to concrete ARNs (DynamoDB tables/indexes, Redis secret, Redis replication group).
- Kept wildcard permissions only where AWS APIs practically require them (`ecr:GetAuthorizationToken`, Chime create calls, CloudWatch `PutMetricData`).
- Attempted to eliminate DynamoDB deprecation warnings; provider currently does not accept the suggested `key_schema` blocks in this configuration, so this remains a known technical-debt item to revisit on provider changes.
