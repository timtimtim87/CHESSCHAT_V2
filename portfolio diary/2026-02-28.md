  - `terraform init -backend=false`
  - `terraform validate` (provider plugin schema failure remains a local environment issue, not a config syntax issue).

### Thought Process and Tradeoffs
- Dependency-cycle prevention was treated as the primary architecture constraint:
  - Previous risk path: ElastiCache SG needed ECS SG, while ECS IAM referenced ElastiCache outputs.
  - Final design removes that cycle by making ECS IAM independent of ElastiCache resource outputs and keeping Redis permissions scoped by deterministic naming patterns.
- ALB integration is one-way from compute to edge (ECS can consume ALB target group/SG outputs, but ALB does not depend on ECS), which keeps graph ordering predictable.
- Kept edge and DNS behind explicit feature toggles so we can apply in controlled phases and avoid premature domain/certificate dependencies.
- Maintained the operational rule: ECR image must exist before ECS service apply to avoid image pull failures in Fargate startup.
