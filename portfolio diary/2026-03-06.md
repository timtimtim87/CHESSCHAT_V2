## Why These Decisions
- Board-first hierarchy is preserved for gameplay clarity and interview-ready architecture narrative.
- Desktop was prioritized for maximum product perception gain in early user playtesting.
- A tokenized style system lowers iteration cost for follow-up feedback cycles and reduces CSS drift.
- Mobile was implemented as safe fallback to keep functionality accessible without delaying desktop polish.

## Tradeoffs / Risks
- Opponent media telemetry (remote mic/camera state) remains limited; this pass focused on local control clarity and overall layout quality.
- Bundle chunk-size warnings remain from existing dependency composition and can be addressed in a later optimization pass.
