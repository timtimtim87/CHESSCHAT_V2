# 2026-02-15

## What We Built
- Phase A network foundation in AWS (dev scope):
  - VPC
  - 3 public subnets
  - 3 private-app subnets
  - 3 private-data subnets
  - Internet Gateway and route tables/associations
  - NAT Gateway strategy implemented with single-NAT active
  - VPC endpoints: S3, DynamoDB, ECR API, ECR DKR, Logs, Secrets Manager
  - VPC Flow Logs to CloudWatch Logs
- Root Terraform module wiring for future layers (ECS, ALB, ElastiCache, DynamoDB, Cognito, Route53, Monitoring).

## How We Built It
- Applied Terraform in the `terraform/` root using dev variables.
- Validated provisioned resources against AWS and captured exact IDs/ARNs.
- Recorded infrastructure outputs and status in `infra/RESOURCE_REGISTRY.md`.
- Updated strategy and execution status in `DOCS/PORTFOLIO_BUILD_PLAYBOOK.md`.

## Thought Process and Tradeoffs
- Deliberately used a 3-AZ subnet layout to show production-grade design and HA awareness.
- Selected single NAT mode first to control cost while still demonstrating a clear upgrade path (`single` -> `per_az`).
- Added VPC endpoints early to improve private connectivity and reduce unnecessary internet egress paths.
- Treated observability as part of foundation (Flow Logs), not as a post-launch afterthought.
- Chose incremental phase-based delivery so each layer is interview-explainable and testable before adding complexity.
